Overview

HT for Web tree tables and topologies, etc., 2D components, no special processing can easily load more than 10,000 Data elements, but for Graph3dView components to load thousands of orders of magnitude, you need to do a special batch processing. The principle of high performance in batch can be that when the graph element is plotted independently has poor performance, the refresh performance of the WebGL is greatly improved when a batch of datas is aggregated into a large model for one time drawing.

Batch processing is equivalent to eating, if a small mouth to eat, bottleneck in the interaction process is too frequent and is slow, if a large spoon can be changed to reduce the interaction process, but WebGL, JavaScript and the browsers are all limited such as memory resources, HT automatically aggregation of all data into a large batch is not realistic, the equivalent of the mouth of our people can not eat a bowl of rice at one time, so we need the user to sum up the different batch scheme of the data to improve rendering performance by their own specific situation of the project.

Data Batch

Currently HT supports the volume of the body portion of the data, and does not support the batch of the label, note and icons parts, so this section describes the data batch of the main data model.

Basic Property

node.setStyle('batch', 'firstBatch');

The above is the simplest batch to set up the way, the same batch name data will automatically aggregate into batch one-time draw. You can experience the performance gap in batches by switching the Enable Batch switch on the previous tool bar.

ht.Default.setBatchInfo('firstBatch', {
    color: 'red',
    brightness: true,
    blend: true
});

Batch also has a lot of reference to control, in most cases need to register the global batch by ht.Default.setBatchInfo function, and then sets the batch batch name for Data. If the batch properties for datas without registered by ht.Default.setBatchInfo are represented to use the HT defaults batch parameter:

  • brightness: The default is false, the select.brightness effect that is selected by the data does not work and needs to be set to true to turn on the highlighting feature
  • blend: The default is false, which means that the datas are not dyed and need to be set to true to turn on the dyeing feature
  • color: The default is undefined, which has a higher priority than the independent graphic data's surface color, so if you set the color all faces of the data will be displayed as the color value

Graph3dView has been turned off by default for batch dyeing and batch brightness, so if you want to enable blend and brightness in batches, you need to enable them by:

  • Enable batch dyeing: graph3dView.setBatchBlendDisabled(false)
  • Enable bulk brightness: graph3dView.setBatchBrightnessDisabled(false)

Texture Property

Batch solves performance problems, but also brings a side effect that reduces or decreases functionality. For example, the current HT a batch only supports a texture, the texture information is decided by registering the volume of the image attribute, so as to hexahedral need different face tile different pictures, you need to combine multiple pictures into a large picture, then, the cube is independently defined for each face corresponding to the uv related parameter to implement.

for (var i = 0; i < row; i++) {
    for (var j = 0; j < column; j++) {
        createNode(startX + gap * j, startZ, size).s({
            'batch': 'uvBatch',                            
            'bottom.uv': [0.25, 0, 0.25, 0.25, 0.5, 0.25, 0.5, 0],
            'left.uv': [0, 0.75, 0, 1, 0.25, 1, 0.25, 0.75],
            'right.uv': [0.5, 0.75, 0.5, 1, 0.75, 1, 0.75, 0.75],                            
            'all.blend': '#00FFFF',
            'top.visible': i % 2 === 0
        });
    }
    startZ -= gap;
}

ht.Default.setBatchInfo('uvBatch', {
    brightness: brightness,
    blend: blend,
    image: 'die',
    topUv: [0.25, 0.5, 0.25, 0.75, 0.5, 0.75, 0.5, 0.5],
    backUv: [0.25, 0.75, 0.25, 1, 0.5, 1, 0.5, 0.75],                    
    frontUv: [0.25, 0.25, 0.25, 0.5, 0.5, 0.5, 0.5, 0.25],
    reverseColor: 'yellow',
    reverseCull: reverseCull,
    reverseFlip: reverseFlip
});

The above example sieves the six faces of the uv information part of the face is global setting when registering batchInfo, the partial faces is based on the style of the datas are setting by *.uv property, the data style configuration of the uv attribute information priority is higher than the registration batch parameter, in general, if the uv information of the batch data faces are the same, through the global configuration is more convenient, do not have to update each data corresponding face properties, if there is a special data that is different from the global can be independently set style corresponding uv information.

  • frontUv: The default is NULL, used to specify the front uv parameter, priority is lower than the data data, specific parameter front.uv and all.uv

Other Properties

In addition to the color, brightness and blend batch parameters described earlier, the following batch reference can be set, these parameters can only be specified in the global register batch, the corresponding property on the specific data is ignored.

  • image: The default is empty, the specified is to represent to display the texture, to show the color for the empty representative
  • light: The default is true, which represents a light effect that can be set to false to turn off the light effect
  • reverseCull: The default is false, determines whether the opposite side is cropped, can be set to true to crop the reverse side
  • reverseFlip: The default is false, whether the reverse side shows positive content, if false then shows reverseColor color
  • reverseColor: The default is #868686, determines the reverse color
  • transparentMask: The default is false, which makes the data do not display in the interface, but can still be clicked to select
  • uvOffset: The default is NULL, which increases the offset on the uv parameter of all sides
  • uvScale: The default is NULL, which is multiplied by the uv parameter of all sides
  • discardSelectable: The default is true, which represents that the opacity of the texture to the excluded part can also be clicked, and can be set to false and the excluded part can not be selected

Transparent Property

Finally there are several transparent related batch parameters, at present, the same amount of support is only opaque objects, or all transparent objects, do not support part of the faces are transparent and part of the face are not transparent of the mixed situation.

  • opacity: The default is undefined, the overall graphics under the 3D transparency, the optional value range 0~1
  • transparent: The default is false, if the texture is transparent, the graphics color is transparent, or set opacity to be transparent, you need to set this argument to true
  • autoSort: The default value is true, the transparent object needs to be drawn from a backcasting way to render color overlay in WebGL system, HT internally defaults to be drawn after the batch transparency is sorted, and in some cases can be set to false to turn off the automatic sorting feature for improved performance

Invalidate Batch

In addition to the side effects of limited function limitations, the frequent updating of datas in batches can also result in poor performance, it is generally recommended that the infrequently changing datas be grouped into the same batch and that the constantly changing datas be grouped in the same batch, and that the following functions can be called before updating if you need to update most of the datas in the same batch. In general, the batch parameters that are registered to the global are less needed to be modified, and if modified, they can be updated by calling the following function to notify the graphical interface:

  • g3d.invalidateBatch(name): Invalid batch data for specified batch name

Line Batch

The Graph3dView section presents two types of wireframe effects:

  • Control the wireframe that is rendered around the cube through wf style type parameters, as described in 3D manual
  • ht.Edge and ht.Polyline renders as wireframe effects, as described in the shape manual

The Graph3dView interior has been automatically batched for all wireframe types, so the user does not have to set the batch on the wireframe.

Switch Batch

As basic properties mentioned, Graph3dView turn off the batch dyeing and batch brightness function by default, batch if need to enable ' blend ' and ' brightness ', set enable by graph3dView.setBatchBlendDisabled(false) and graph3dView.setBatchBrightnessDisabled(false).

Graph3dView also has a batchColorDisabled attribute, which defaults to false to enable batch color, when the system scene does not need the batch, and does not use the wireframe effect, can turns off batch color by graph3dView.setBatchColorDisabled(true), which helps improve performance and leaves more resources to allow more light to be loaded light effect.
