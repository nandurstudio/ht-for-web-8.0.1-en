Alarm

The alarm is used to describe the abnormal and error information of the network element in the telecommunications network management. The HT in the network element has the alert status attribute (ht.Data#getAlarmState(), type ht.AlarmState), used to describe the alarm information of the network element. The alert status records the new alarm and the confirmation alarm corresponds to each alarm level (ht.AlarmSeverity) number of alarms.

Alarm Level

Alarm level (ht.AlarmSeverity) describes the severity of the alarm and has the following attributes:

  • value Value (number), used to sort alarm levels
  • name Name (string), used to uniquely identify the alarm level
  • nickName Nickname, typically a character used to display a short warning level name on the alarm bubbling
  • color Color (color), used to fill background of the alarm bubbling
  • displayName Display name (string), used to describe the alarm level in detail

Default Alarm Level

HT predefined the following warning levels:

  • AlarmSeverity.CRITICAL Serious alarm: {value: 500, name: 'Critical', nickName: 'C', color: '#FF0000'}
  • AlarmSeverity.MAJOR Main alarm: {value: 400, name: 'Major', nickName: 'M', color: '#FFA000'}
  • AlarmSeverity.MINOR Secondary alarm: {value: 300, name: 'Minor', nickName: 'm', color: '#FFFF00'}
  • AlarmSeverity.WARNING Warning alarm: {value: 200, name: 'Warning', nickName: 'W', color: '#00FFFF'}
  • AlarmSeverity.INDETERMINATE Unknow alarm: {value: 100, name: 'Indeterminate', nickName: 'N', color: '#C800FF'}
  • AlarmSeverity.CLEARED Cleared alarm: {value: 0, name: 'Cleared', nickName: 'R', color: '#00FF00'}

Custom Alarm Level

In addition to the default alert level provided by HT, you can add custom alert levels in the following ways:

AlarmSeverity.add(value, name, nickName, color, displayName)
  • value Value (number), used to sort alarm levels
  • name Name (string), used to uniquely identify the alarm level
  • nickName Nickname, typically a character used to display a short warning level name on the alarm bubbling
  • color Color (color), used to fill background of the alarm bubbling
  • displayName Display name (string), used to describe the alarm level in detail
  • return ht.AlarmSeverity

Example: 

var trivial = ht.AlarmSeverity.add(50, 'Trivial', 'T', '#20B2AA');
console.assert(trivial === ht.AlarmSeverity.getByName('Trivial'),
	"trivial === ht.AlarmSeverity.getByName('Trivial')");

Alarm Status

Alarm status (ht.AlarmState) is the attribute of the network element in HT (Data#getAlarmState()). Alarm points new alarm (New Alarm) and acknowledged alarm (Acknowledged Alarm), the alert status records the new alarm and the acknowledged alarm corresponds to each alarm level (ht.AlarmSeverity) number of alarms. In addition, the alert status also records the level of the propagation alarm (propagation alarm refers to the highest alarm level for all children of the network element).

If there is a new alarm, GraphView style note2 is replaced by the new alarm message, the fill color is the highest new alarm level color, the text is the highest new alarm level number and nickname:

ht.GraphView.prototype.getNote2 = function (data) {
	var severity = data.getAlarmState().getHighestNewAlarmSeverity();
	if (severity) {
		var label = data.getAlarmState().getNewAlarmCount(severity) + severity.nickName;
		if (data.getAlarmState().hasLessSevereNewAlarms()) {
			label += "+";
		}
		return label;
	}
	return data.getStyle('note2');
};
ht.GraphView.prototype.getNote2Background = function (data) {
    var severity = data.getAlarmState().getHighestNewAlarmSeverity();
    if (severity) {
        return severity.color;
    }
    return data.getStyle('note2.background');
};

If there is a acknowledged alarm, the fill color of the data changes to the highest new or confirmed warning level:

ht.GraphView.prototype.getBodyColor = function (data) {
	var severity = data.getAlarmState().getHighestNativeAlarmSeverity();
	if (severity) {
		return severity.color;
	}
	return data.getStyle('body.color');
};

If there is a propagation alarm, the grid element's border color becomes the color of the propagation alarm:

ht.GraphView.prototype.getBorderColor = function (data) {
	var severity = data.getAlarmState().getPropagateSeverity();
	if (severity) {
		return severity.color;
	}
	return data.getStyle('border.color');
};

Alarm Propagation

In HT, the alarm spreads ht.AlarmStatePropagator will transmit the alarm of the underlying network element to the upper network element, such as the highest level warning of all children in Group or SubGraph will be propagated to Group or SubgGraph. The default alert propagation is not turned on and needs to be called ht.DataModel#setEnableAlarmStatePropagator(true) turns on alarm propagation.

Custom alarm propagation

The default alert propagation is spread from child to father and can be rewritten ht.AlarmStatePropagator propagateToTop method changes the propagation mechanism, and the following example shows the alarm propagating along the host:

First close the alarm propagation of the data container (ht.DataModel#setEnableAlarmStatePropagator(false)), and then create a new ht.AlarmStatePropagator object, then rewrite AlarmStatePropagator#handleDataPropertyChange(e) and AlarmStatePropagator#propagateToTop(data) method:

propagator.handleDataPropertyChange = function (e) {
	ht.AlarmStatePropagator.prototype.handleDataPropertyChange.call(this, e);
	if (e.property === 'host') {
    	var oldHost = e.oldValue;
		if (oldHost) {
			this.propagate(oldHost);
		}
		this.propagate(e.data);
	}
};
propagator.propagateToTop = function (data) {
	this.propagateToHost(null, data);
	while (data && data.getHost()) {
		this.propagateToHost(data, data.getHost());
		data = data.getHost();
	}
};
propagator.propagateToHost = function (attach, host) {
	var result = null;
	if (host.getAttaches()) {
		host.getAttaches().each(function (attach) {
			var severity = attach.getAlarmState().getHighestOverallAlarmSeverity();
			if (ht.AlarmSeverity.compare(severity, result) > 0) {
				result = severity;
			}
			});
		}
		host.getAlarmState().setPropagateSeverity(result);
	};

Custom Alarm Statistics Panel

The following example demonstrates the use of vector to implement alarm statistics panel:

Alarm on the Tree

The icon fill color of the node on the tree is the highest new or acknowledgee warning level in the data in default, and if there is a propagation alarm, the tree node icon adds the color to the border of the propagation alert level color:

ht.widget.TreeView.prototype.getIcon = ht.widget.TreeTableView.prototype.getIcon = function (data) {
	return '__alarmIcon__';
};
ht.Default.setImage('__alarmIcon__', {
    width: 18,
    height: 18,
    comps: [
        {
            type: 'image',
            name: {
                func: function (data) { return data.getIcon(); }
            },
            color: {
                func: function (data) {
                    var severity = data.getAlarmState().getHighestNativeAlarmSeverity();
                    if (severity) {
                        return severity.color;
                    }
                    return data.getStyle('body.color');
                }
            },
            rect: [1, 1, 16, 16]
        },
        {
            type: 'rect',
            borderWidth: 2,
            borderColor: {
                func: function (data) {
                    var severity = data.getAlarmState().getPropagateSeverity();
                    if (severity) {
                        return severity.color;
                    }
                    return data.getStyle('border.color');
                }
            },
            visible: {
                func: function (data) {
                    return !!data.getAlarmState().getPropagateSeverity() || !!data.getStyle('border.color');
                },
            },
            rect: [1, 1, 16, 16]
        }
    ]
});
